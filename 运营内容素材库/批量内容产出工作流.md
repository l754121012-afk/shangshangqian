# 批量内容产出工作流

## 一、工作流总览

所有需要反复产出的运营内容，统一按以下 5 步执行：

```
1. 确定主题/方向（5分钟）
2. 一次性产出批量内容（借助 AI 或模板）
3. 人工审核 + 调优（10-20分钟）
4. 存为 JSON 文件，代码直接读取
5. 上传 Git，记录版本
```

---

## 二、各内容类型的产出方法

### 2.1 每日金句（每月一次，30句）

**触发条件**：每月最后一天或月初第一天

**工作流**：
1. 打开 `daily-quotes.json`，查看本月数据表现（哪类主题最受欢迎）
2. 确定下月主题侧重（如：9月侧重开学/成长，12月侧重复盘/新年）
3. 用以下 Prompt 让 AI 批量产出：

```
请为"今日上上签"小程序产出下个月（XX月）的30条每日金句。
要求：
- 每条15-30字，温暖、有力量、不鸡汤
- 主题是每日心理仪式感，不是算命
- 覆盖以下主题（可重复）：自我肯定、行动力、情绪管理、周末、开始、复盘
- 输出格式：JSON 数组，字段：day, quote, theme
- 不要出现"命运""天意"等词汇
- "转运"类表达改为"换一种视角"
```

4. 人工审核，替换掉不合适的内容
5. 保存到 `daily-quotes.json`，更新 version 字段
6. Git 提交

---

### 2.2 分享卡片模板（每季度一次，2-3套）

**触发条件**：分享率数据下降 / 新节日临近 / 季度更新

**工作流**：
1. 分析当前各模板的分享率数据
2. 确定新模板方向（如：开学季、双十一、年末复盘）
3. 设计新模板的视觉风格（可用 AI 出参考图）
4. 写入 `share-templates.json`
5. 前端代码中注册新模板
6. Git 提交

---

### 2.3 节日专题内容（提前 2 周准备）

**触发条件**：节日日期前 14 天

**工作流**：
1. 查看 `holiday-events.json`，确认即将到来的节日
2. 为每个节日设计 1-2 张专属签卡
3. 设计节日专属视觉（背景色、装饰元素）
4. 写入 `holiday-events.json`
5. 前端代码中配置节日自动切换逻辑
6. Git 提交

---

## 三、内容审核 checklist

所有产出内容在入库前必须检查：

- [ ] 无"算命""命运""天意"等词汇
- [ ] 无承诺真实财富/运势改变的表达
- [ ] "转运"类表达已改为"换一种视角"
- [ ] 负面签附带消解方式
- [ ] 微行动是当天可完成的具体小事
- [ ] 广告入口不出现在负面/焦虑场景中
- [ ] 内容长度符合字段限制

---

## 四、版本管理

每次批量产出后，务必：

1. 更新 JSON 文件中的 `version` 字段
2. Git 提交，commit message 格式：`content: XXX内容更新（月份/版本）`
3. 在 `上线前运营准备清单.md` 中更新状态
